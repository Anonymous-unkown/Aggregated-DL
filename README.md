# Overview
This framework addresses a core challenge in computational genomics: classifying full-length sequences (e.g., 200 bp) when only a small number of labeled examples are available (e.g., 50–100 sequences per class). The approach is based on a data augmentation strategy that generates overlapping augmented subsequences, which has already been developed by Abbasi-Vineh et al., 2025.
The approach combines:
•	Overlapping subsequence augmentation — each 200 bp sequence is padded and decomposed into 240 overlapping 40-nt subsequences, dramatically expanding the training data while preserving full-length coverage.
•	A hybrid CNN–LSTM–Attention–Residual model — hierarchically extracts local motif features (CNN), models positional dependencies (bidirectional LSTM), and dynamically weights informative positions (attention mechanisms).
•	Coverage-aware feature aggregation — subsequence-level predictions and features are aggregated back to the original full-length sequence using binary masking and attention-weighted averaging, ensuring padded regions do not bias classification.
•	A dual hybrid loss function — jointly optimizes subsequence-level and sequence-level cross-entropy losses during training (weighting factor α = 0.7).
The framework was evaluated on three independent genomic datasets covering evolutionarily divergent organisms, chloroplast genomes, and regulatory sequences with and without Shine-Dalgarno motifs.

Embeddings
Pre-computed CNN embeddings generated by the trained model for all three datasets are provided in the embeddings/ directory as NumPy .npy files. Each embedding file contains the aggregated feature vectors (extracted after the fc1–fc3 fully connected layers, prior to the final classification layer) for all original full-length sequences in that dataset.
These embeddings can be loaded directly for downstream analyses such as similarity analysis, clustering, or transfer learning, without re-running the full training pipeline:
import numpy as np

embeddings = np.load('embeddings/dataset1_embeddings.npy', allow_pickle=True).item()
# embeddings is a dict: {sequence_id: feature_vector (numpy array)}

Usage
All steps of the pipeline are implemented in the single notebook Primary_Scripts_for_Aggregated_DL.ipynb. The notebook is organized into the following sequential sections:
Step 1 — Data Loading and Preparation
Load your FASTA sequences and assign class labels. Place your dataset files in the datasets/ directory (or update the file paths in the notebook). The notebook expects sequences of equal length (200 bp); no further pre-processing is required.
Step 2 — Overlapping Subsequence Augmentation
Each 200 bp sequence is padded with 40-nt poly-'N' at both ends (producing a 280-nt padded sequence), then decomposed into 240 overlapping 40-nt subsequences. This step is handled automatically by the augmentation function in the notebook.
Step 3 — One-Hot Encoding and Masking
Each 40-nt subsequence is encoded as a 5-channel one-hot matrix (A, T, C, G, N). Binary masks are generated simultaneously to track the positions of valid nucleotides versus padding.
Step 4 — Model Training (3-Fold Cross-Validation)
Run the training cell to train the CNN–LSTM–Attention–Residual model using the dual hybrid loss function. Training parameters:
Parameter	Value
Epochs	100
Batch size	256
Optimizer	AdamW (weight decay = 0.01)
Scheduler	OneCycleLR
Loss weight (α)	0.7
Cross-validation	Stratified 3-fold (80/20 train/test, 20% validation)
Step 5 — Subsequence-Level Evaluation
Evaluate the model directly on held-out 40-nt subsequences. Classification reports, confusion matrices, ROC curves, and PR curves are generated automatically.
Step 6 — Original Sequence-Level Evaluation
Aggregate subsequence features using attention-weighted averaging (Equation 4 in the paper) to produce a single prediction per original 200 bp sequence. Full classification metrics are reported at this level as well.
Step 7 — Saliency Mapping and Interpretability
Compute gradient-based saliency maps to identify nucleotide positions critical for classification. Class-averaged saliency profiles are plotted against the 200 bp original sequence coordinates.
Step 8 — Saving Results and Embeddings
The trained model state, metadata, sequence mappings, and embeddings are saved automatically at the end of the notebook.

Model Architecture
The model consists of the following components in sequence:
Input (40-nt one-hot, 5 channels)
    │
    ▼
CNN Block 1 — Conv1D (128 filters, k=3) + BN + MaxPool + Dropout(0.3) + Residual + CNN-Attention
    │
    ▼
CNN Block 2 — Multi-kernel Conv1D (k=3,5,7; 256 filters each → 768 ch) + BN + MaxPool + Dropout(0.5) + Residual + CNN-Attention
    │
    ▼
CNN Block 3 — Multi-kernel Conv1D (k=3,5,7; 512 filters each → 1536 ch) + BN + MaxPool + Dropout(0.3) + Residual + CNN-Attention
    │
    ▼
Positional Encoding
    │
    ▼
Bidirectional LSTM (2 layers, hidden=256 per direction, total=512) + Dropout(0.3)
    │
    ▼
LSTM-Attention → Context vector + Attention weights (stored for aggregation)
    │
    ▼
FC1 (256 units) + BN + Dropout(0.3) + ReLU
FC2 (512 units) + Dropout(0.5) + ReLU
FC3 (512 units) + Dropout(0.3) + ReLU   ← Feature vector extracted here for embeddings
    │
    ▼
FC4 → Softmax → Class probabilities
For original sequence-level classification, the LSTM-Attention weights from all 240 subsequences are normalized and used to compute a weighted average of the FC3 feature vectors, which is then passed through FC4.
